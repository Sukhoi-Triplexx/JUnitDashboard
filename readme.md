# JUnit Enterprise Viewer

Система для автоматизированного анализа, хранения и визуализации результатов JUnit-тестов. Поддерживает ручную загрузку XML-файлов, автоматический мониторинг директории, уведомления в Telegram, историю по датам, детальные отчёты с графиками и экспорт ошибок.

## Функциональные возможности

- Загрузка одного или нескольких JUnit XML-файлов через веб-интерфейс.
- Загрузка ZIP-архива с XML-файлами.
- Автоматическое сканирование заданной папки (в том числе сетевых дисков) с интервалом в секундах.
- Отправка уведомлений о новых отчётах в Telegram (опционально).
- Хранение всех отчётов в SQLite с группировкой по датам.
- Дашборд с агрегированной статистикой по дням, графиками распределения и трендом проходного балла.
- Детальная страница отчёта:
  - Сводные карточки (passed, failed, errors, total tests, pass rate, files).
  - Круговая диаграмма результатов.
  - Столбчатая диаграмма проходного балла по файлам.
  - Поиск по названию теста.
  - Раскрывающиеся блоки с деталями тестов и полным stack trace.
  - Таблица ошибок (failed/error) с возможностью разворачивания строк для просмотра полного сообщения и traceback.
  - Экспорт всех ошибок в CSV или JSON.
- Управление отчётами: удаление одного отчёта или всех отчётов с подтверждением.
- Тёмная OLED-тема интерфейса, адаптивная вёрстка.
- Настройки: включение/отключение авто-мониторинга, указание папки, интервал сканирования, настройки Telegram (токен, chat_id, уведомления).

## Технологический стек

- Backend: Python 3.11+, Flask
- База данных: SQLite
- Фронтенд: HTML, CSS (адаптивный, тёмная тема), JavaScript, Chart.js
- Асинхронный мониторинг: потоки Python (threading)

  ## Конфигурация

Все настройки хранятся в `config.json` и редактируются через веб-интерфейс (страница `/settings`):

- `auto_monitoring` – включение автоматического сканирования папки.
- `watched_folder` – путь к наблюдаемой папке (поддерживаются абсолютные и относительные пути, в том числе сетевые UNC).
- `scan_interval` – интервал сканирования в секундах (по умолчанию 30).
- `telegram_enabled` – включение отправки уведомлений.
- `telegram_token` – токен бота, полученный от @BotFather.
- `telegram_chat_id` – идентификатор чата или пользователя.
- `notify_on_new` – уведомлять о новых отчётах.

## Использование

### Главная страница

- Перетащите XML-файлы в область загрузки или выберите их через кнопку "Выбрать файлы".
- Поддерживается загрузка нескольких файлов одновременно.
- Также можно загрузить ZIP-архив, содержащий XML-файлы.

### Дашборд

- Показывает список дней, в которые были загружены отчёты.
- При выборе дня отображаются:
  - Сводные метрики за день (passed, failed, errors, total tests, avg pass rate, total time).
  - Круговая диаграмма распределения результатов по всем дням (накопительная).
  - Линейный график тренда проходного балла за последние 14 дней.
  - Список отчётов за выбранный день с возможностью перехода к детальному просмотру.
  - Кнопка удаления отдельного отчёта и кнопка удаления всех отчётов (с подтверждением).

### Детальный отчёт

- Сводные карточки по текущему отчёту.
- Круговая диаграмма и столбчатая диаграмма для данного отчёта.
- Поиск по названию теста.
- Список файлов (один или несколько) с возможностью раскрытия.
- Внутри каждого файла – сьюиты и тест-кейсы.
- Для упавших тестов (`failed` или `error`) отображается сообщение и полный stack trace.
- **Таблица ошибок** (кнопка "SHOW ERRORS TABLE"):
  - Содержит все тесты со статусом `failed` или `error`.
  - При клике на строку разворачивается блок с полным текстом ошибки и stack trace.
  - Возможность экспорта таблицы в CSV или JSON (кнопки "EXPORT ERRORS TO CSV" и "EXPORT ERRORS TO JSON").
- Кнопка "OPEN IN NEW TAB" для открытия текущего отчёта в новой вкладке.

### Настройки

- Включение/отключение авто-мониторинга.
- Путь к папке для отслеживания.
- Интервал сканирования.
- Параметры Telegram-бота (токен, chat_id, уведомления).
- Кнопка принудительного сканирования.

## Структура базы данных

`junit_reports.db` (SQLite) содержит две таблицы:

- `report_groups` – хранит группы отчётов (одиночные или множественные загрузки):
  - `group_id` – уникальный идентификатор.
  - `created_date`, `created_date_str` – дата создания.
  - `group_data` – JSON с полными данными отчёта.
  - `file_count`, `combined_summary` – агрегированная статистика.
  - `is_auto` – флаг автоматической обработки.
  - `display_name` – отображаемое имя (имя файла или "Batch (N files)").

- `processed_files` – хранит хеши обработанных файлов для предотвращения повторной обработки при авто-мониторинге.

## API эндпоинты (используются фронтендом)

- `GET /` – главная страница.
- `GET /dashboard` – страница дашборда.
- `GET /settings` – страница настроек.
- `POST /upload` – загрузка одного или нескольких XML-файлов.
- `POST /upload/zip` – загрузка ZIP-архива.
- `GET /report/<group_id>` – страница детального отчёта.
- `GET /api/report/<group_id>` – получение JSON с данными отчёта.
- `GET /api/dates` – список дат, для которых есть отчёты.
- `GET /api/reports/by-date/<date>` – список отчётов за конкретную дату.
- `GET /api/config` – получение текущей конфигурации.
- `POST /api/config` – сохранение конфигурации.
- `POST /api/scan` – ручной запуск сканирования папки.
- `DELETE /api/delete-report/<group_id>` – удаление одного отчёта.
- `DELETE /api/delete-all-reports` – удаление всех отчётов.
- `POST /api/check-path` – проверка доступности пути (для сетевых дисков).

## Особенности работы с сетевыми дисками

- При указании UNC-пути (например, `\\server\share\folder`) приложение нормализует его для работы в Windows.
- Если путь недоступен, в консоли выводится сообщение с предложением проверить `net use`.
- Для монтированных дисков (`Z:`) путь проверяется стандартными средствами `os.path.exists`.

## Устранение неполадок

- **Ошибка импорта Flask** – установите Flask вручную с помощью скачанных `.whl` файлов.
- **Ошибка "no such table: display_name"** – удалите `junit_reports.db` и перезапустите приложение (структура БД создастся заново).
- **Сетевой диск не виден** – выполните в командной строке `net use` для проверки подключения.
- **Мониторинг не работает** – убедитесь, что в настройках включен флаг `auto_monitoring` и папка существует/доступна.
