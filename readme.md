# VUnit

Система для автоматизированного анализа, хранения и визуализации результатов JUnit-тестов. Поддерживает ручную загрузку XML-файлов, автоматический мониторинг директории (включая сетевые диски), уведомления в Telegram и Email, историю по датам, детальные отчёты с графиками, экспорт ошибок, REST API и переключение между SQLite и PostgreSQL.

## Функциональные возможности

### Загрузка и обработка
- Загрузка одного или нескольких JUnit XML-файлов через веб-интерфейс
- Загрузка ZIP-архива с XML-файлами
- Поддержка файлов до 300MB
- Потоковая обработка больших файлов (>10MB)

### Автоматизация
- Автоматическое сканирование заданной папки (в том числе сетевых дисков) с настраиваемым интервалом
- Отправка уведомлений о новых отчётах в Telegram и Email
- REST API для CI/CD интеграции

### Аналитика и визуализация
- Дашборд с агрегированной статистикой по дням
- Круговая диаграмма распределения результатов
- Линейный график тренда проходного балла за 14 дней
- Метрики за день

### Детальный отчёт
- Сводные карточки с ключевыми метриками
- Круговая и столбчатая диаграммы
- Поиск по названию теста
- Раскрывающиеся блоки с деталями тестов и полным stack trace
- Таблица ошибок с разворачиванием строк
- Экспорт ошибок в CSV или JSON

### Управление данными
- Хранение отчётов в SQLite или PostgreSQL
- Автоматические бэкапы базы данных
- Экспорт/импорт всех данных в JSON
- Очистка старых отчётов (30+ дней)

### Интерфейс
- Тёмная OLED-тема с возможностью переключения на светлую
- Адаптивная вёрстка для мобильных устройств
- Анимированные переходы и уведомления

## Технологический стек

- Backend: Python 3.11+, Flask
- База данных: SQLite / PostgreSQL
- Фронтенд: HTML, CSS, JavaScript, Chart.js
- Иконки: Font Awesome
- Контейнеризация: Docker

## Установка и запуск

1. Убедитесь, что установлен Python 3.11 или выше
2. Склонируйте репозиторий или скопируйте файлы проекта
3. Установите зависимости:
Конфигурация
Все настройки хранятся в config.json и редактируются через веб-интерфейс (страница /settings):

auto_monitoring – включение автоматического сканирования папки

watched_folder – путь к наблюдаемой папке (поддерживаются абсолютные и относительные пути, в том числе сетевые UNC)

scan_interval – интервал сканирования в секундах (по умолчанию 30)

telegram_enabled – включение отправки уведомлений в Telegram

telegram_token – токен бота, полученный от @BotFather

telegram_chat_id – идентификатор чата или пользователя

email_enabled – включение email уведомлений

smtp_server – SMTP сервер

smtp_port – порт SMTP

email_from – email отправителя

email_to – email получателя

postgres_enabled – использование PostgreSQL вместо SQLite

pg_host, pg_port, pg_database, pg_user, pg_password – параметры подключения к PostgreSQL

Настройка PostgreSQL
Быстрый запуск через Docker
bash
docker run -d \
  --name postgres \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres123 \
  -e POSTGRES_DB=vunit \
  -p 5432:5432 \
  postgres:15
Подключение к VUnit
Откройте http://localhost:5000/settings

Включите тумблер "Use PostgreSQL"

Заполните поля подключения

Нажмите "TEST CONNECTION"

Сохраните настройки и перезапустите VUnit

REST API
Базовый URL: http://localhost:5000/api/v1

Управление ключами
GET /keys – список API ключей

POST /keys/generate – создать новый ключ

DELETE /keys/{id} – удалить ключ

Работа с отчётами
GET /health – health check

GET /reports – список отчётов (пагинация)

GET /reports/{id} – детали отчёта

GET /reports/{id}/summary – только сводка

POST /upload – загрузить XML/ZIP

DELETE /reports/{id} – удалить отчёт

GET /stats – общая статистика

GET /search?q= – поиск по тестам

Аутентификация: заголовок X-API-Key

Пример использования
bash
# Сгенерировать ключ
curl -X POST http://localhost:5000/api/v1/keys/generate

# Загрузить отчёт
curl -X POST http://localhost:5000/api/v1/upload \
  -H "X-API-Key: your_key" \
  -F "file=@report.xml"

Docker
Docker Compose
yaml
version: '3.8'
services:
  vunit:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - ./watched_reports:/app/watched_reports
      - ./backups:/app/backups
    restart: unless-stopped
Запуск: docker-compose up -d

Особенности работы с сетевыми дисками
Поддерживаются UNC-пути (например, \\server\share\folder)

Поддерживаются монтированные диски (например, Z:\reports)

Если путь недоступен, в консоль выводится сообщение с рекомендацией проверить net use

Устранение неполадок
Ошибка импорта Flask – установите Flask вручную с помощью скачанных .whl файлов

Ошибка "no such table: display_name" – удалите junit_reports.db и перезапустите приложение

Сетевой диск не виден – выполните в командной строке net use для проверки подключения

Мониторинг не работает – убедитесь, что в настройках включен флаг auto_monitoring и папка существует/доступна

PostgreSQL не подключается – проверьте что сервер запущен и параметры подключения указаны верно
